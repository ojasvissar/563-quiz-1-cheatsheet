Unsupervised Learning — The Big Picture

What is it and why do we need it?

In supervised learning you have data + labels (like photos + whether they're cats or dogs). You train the model to predict the label.

In unsupervised learning you only have data — no labels. You want the machine to discover hidden structure on its own.

🌍 Real-Life Analogy
A librarian sorts thousands of books without a guide. She notices patterns: these books all have maps → geography; those have equations → math. Nobody told her the categories — she found them. That's unsupervised learning.

The Two Main Tasks

🔵 Clustering
Goal: Group similar things together
Methods: K-Means, DBSCAN, GMM, Hierarchical
Output: Cluster labels for each point
🟡 Dimensionality Reduction
Goal: Compress many features into fewer
Methods: PCA, NMF, SVD, Autoencoders
Output: A smaller representation of data

Applications from the Cheatsheet

ApplicationWhat it does
Customer SegmentationGroup customers by purchasing behavior for targeted marketing
Document ClusteringGroup news articles by topic (sports, politics, finance...)
Image CompressionRepresent images with fewer numbers (PCA/NMF)
Anomaly DetectionFind unusual data points (fraudulent transactions)
VisualizationCompress high-dimensional data to 2D for plotting
Feature EngineeringCreate better input features for supervised models

K-Means Clustering — Deep Dive

The Core Concept

K-Means splits data into K clusters. Each cluster is defined by its centroid (mean center). Each point belongs to the cluster whose centroid is closest.

🏙️ Fire Station Analogy
Place K fire stations across a city. Every house is served by its nearest station. K-Means: (1) places stations, (2) assigns each house to nearest station, (3) moves stations to the average position of all their houses, (4) repeats until stable. The fire stations are the centroids!

The Full Algorithm (Step by Step)

1
Pick K — decide how many clusters you want. This is a hyperparameter you choose.
2
Initialize centroids — place K starting centroids. Either random or K-Means++ (smarter, spreads them out).
3
Assignment step — assign each data point to its nearest centroid (using Euclidean distance).
4
Update step — move each centroid to the mean (average position) of all points assigned to it.
5
Repeat steps 3 and 4 until assignments don't change (convergence).
⚠️ Convergence
K-Means always converges (stops changing) but may get stuck at a local minimum — a decent solution that isn't the best possible. This is why sklearn runs it multiple times with different initializations (n_init=10 by default) and keeps the best result.

The Objective Function — What K-Means Minimizes

K-Means tries to minimize inertia (also called within-cluster sum of squares, WCSS):

Inertia = Σ Σ || xᵢ - μₖ ||²

Where μₖ is the centroid of cluster k and xᵢ is each point in that cluster. Lower inertia = tighter, more compact clusters.

📏 Inertia Analogy
Imagine measuring how far everyone lives from their nearest fire station. Add up all those distances squared. That's inertia. K-Means tries to place fire stations to minimize this total "travel distance."

sklearn Code — Full Reference

from sklearn.cluster import KMeans

# Create and fit
km = KMeans(
    n_clusters=4,      # K — number of clusters (YOU choose)
    n_init=10,         # Run 10 times, keep best (avoids local minima)
    random_state=42    # For reproducibility
)
labels = km.fit_predict(X)  # Fit and get labels in one step

# Key attributes (things you can access AFTER fitting)
km.cluster_centers_   # Shape (K, d) — the K centroids
km.labels_            # Cluster assignment for each training point
km.inertia_           # Final inertia value (lower = better)
km.n_iter_            # How many iterations it took

# To predict new points
km.predict(X_new)     # Uses existing centroids — no re-fitting!

Parameters vs. Hyperparameters

TypeWhatHow set
Hyperparametern_clusters (K)YOU set it before training

💡 Intuition
Hyperparameter = dial you turn before pressing "go". Parameter = the model figures it out itself. For K-Means, you pick K, the model figures out WHERE the K centers go.

Choosing K — The Elbow & Silhouette Methods

Elbow Method (Inertia Plot)

Run K-Means for K=2,3,4,5,6,7... and plot inertia. It always decreases as K grows (more clusters = smaller distances). Look for the "elbow" — where the decrease suddenly slows. That K is usually optimal.

💪 Elbow Analogy
Think of diminishing returns. At K=4, adding a 5th cluster cuts inertia a lot. At K=8, adding a 9th barely helps. The elbow is where adding more stations stops being worth it.
⚠️ Limitation
The elbow is often not obvious — the curve can be smooth with no clear kink. Use silhouette score as a backup!

K-Means++ Initialization

Instead of random initialization, K-Means++ picks centroids that are spread far apart. The 2nd centroid is chosen with probability proportional to distance from the 1st, and so on. This gives a better starting point and speeds up convergence. sklearn uses this by default.

When K-Means Works & When It Fails

✅ Works Great When

  • Clusters are roughly spherical (round blobs)
  • Clusters are similar in size
  • Clusters have similar density
  • You need speed — K-Means is very fast

❌ Fails When

  • Non-spherical shapes — rings, crescents, long thin clusters
  • Very different densities — one tight cluster, one spread out
  • Many outliers — outliers drag centroids toward them
  • Overlapping clusters — hard boundaries can't handle soft overlaps
🚨 Key Limitation
K-Means draws linear decision boundaries — straight lines (or planes in higher dimensions) between clusters. It literally cannot draw a curved boundary. So concentric rings? Hopeless.

The "Chicken and Egg" Problem — Initialization Sensitivity

K-Means has a circular dependency: you need centroids to assign points, but you need assigned points to compute centroids. Bad initialization can lead to bad final clusters. This is why you run it multiple times (n_init).

🥚 Chicken and Egg
To know where the fire stations should be, you need to know which houses they serve. But to know which houses they serve, you need to know where the stations are. K-Means breaks this loop by starting with a guess and iterating.

Silhouette Score — Measuring Cluster Quality

What Is It?

The silhouette score measures how well each point fits its own cluster compared to neighboring clusters. It's a way to evaluate clustering quality without knowing the true labels.

s(i) = (b(i) − a(i)) / max(a(i), b(i))

Where:

  • a(i) = average distance from point i to all OTHER points in its cluster ("how tight is my own cluster?")
  • b(i) = average distance from point i to points in the nearest OTHER cluster ("how far am I from my neighbors?")

Interpreting the Score

Score RangeMeaning
Close to +1Point is well-matched to its cluster and far from others — great!
Close to 0Point is on the boundary between clusters — ambiguous
Close to −1Point might be in the wrong cluster — bad
🏡 Neighborhood Analogy
Imagine you live in a neighborhood. a(i) = average distance to your neighbors (are you close to your own community?). b(i) = average distance to the nearest different neighborhood. If you're much closer to your own neighbors than to outsiders → high silhouette (you clearly belong here). If you're equally close to two neighborhoods → silhouette near 0 (you're on the border). 
from sklearn.metrics import silhouette_score

score = silhouette_score(X, labels)  # Overall average
# Higher = better clustering
# Use this to compare different K values!
💡 Use Both Methods
Elbow method (inertia) + Silhouette score together give you a better picture. If elbow suggests K=4 and silhouette peaks at K=4, you're confident!

DBSCAN — Density-Based Clustering

The Core Idea

DBSCAN doesn't use centroids. Instead, it defines clusters as dense regions of points separated by sparse regions. Points in sparse regions are noise.

🌆 City Population Map
Look at a population density map. Dense urban areas = clusters. Sparse countryside = noise. DBSCAN would identify Manhattan, Brooklyn, and the Bronx as separate clusters (if they had a gap between them), and a lone farmhouse in the countryside as noise. Crucially, the "city" can be any shape — no need for it to be a circle!

How DBSCAN Works — The Algorithm

Two hyperparameters control everything:

  • eps (ε) — the neighborhood radius. "How far away can a neighbor be?"
  • min_samples — minimum points within eps radius to be considered a core point

Three Types of Points

TypeDefinitionAnalogyLabel
Core PointHas ≥ min_samples neighbors within epsBusy city center — lots of neighborsGets its own cluster
Border PointNot core itself, but within eps of a core pointCity outskirts — near a busy areaJoins core's cluster
Noise PointNeither core nor border — isolatedLone farmhouse in countryside−1 (noise)
Each point: look around in radius eps If ≥ min_samples neighbors → CORE POINT ★ If < min_samples but near a core → BORDER POINT ◉ Otherwise → NOISE (label = -1) ✕ Clusters = chains of core points that can "reach" each other Border points join the cluster of their nearest core point

Effect of Hyperparameter Changes

ChangeEffect on core pointsEffect on clusters
↑ eps (larger radius)More neighbors → more core pointsBigger, fewer clusters; less noise
↓ eps (smaller radius)Fewer neighbors → fewer core pointsSmaller, more clusters; more noise
↑ min_samples (need more)Harder to qualify → fewer core pointsMore conservative clusters; more noise
↓ min_samples (need fewer)Easier to qualify → more core pointsMore generous clusters; less noise
⚠️ Quiz Alert
Increasing min_samples AND decreasing eps both make it harder to be a core point → FEWER core points, NOT more. Both changes work in the same direction!

Parameters vs. Hyperparameters for DBSCAN

  • Hyperparameters eps, min_samples, distance metric
  • Parameters None! DBSCAN is fully non-parametric. It stores nothing — applies rules fresh each time.

How to Pick eps — The k-Distance Plot

For each point, find its k-th nearest neighbor distance (where k = min_samples). Sort these distances and plot them. Look for the "knee" in the curve — that's your eps.

from sklearn.neighbors import NearestNeighbors

nbrs = NearestNeighbors(n_neighbors=min_samples).fit(X_scaled)
distances, _ = nbrs.kneighbors(X_scaled)
k_distances = np.sort(distances[:, -1])  # k-th neighbor distance

plt.plot(k_distances)
plt.ylabel("Distance to k-th neighbor")
# Look for the "elbow/knee" — that's your eps!
💡 Must Scale!
DBSCAN uses distances. If feature 1 is in meters and feature 2 is in kilometers, distances are meaningless. Always StandardScaler() your data before DBSCAN!

DBSCAN Strengths and Weaknesses

✅ Strengths

  • Any cluster shape — rings, crescents, S-curves
  • No K needed — finds number of clusters automatically
  • Noise detection — explicitly labels outliers as −1
  • Robust to outliers — they don't distort cluster shapes

❌ Weaknesses

  • Varying density — one eps can't fit tight AND sparse clusters
  • Scaling required — eps is distance-sensitive
  • Hard to tune eps — requires domain knowledge or k-distance plot
  • High dimensions — distances become meaningless (curse of dimensionality)

Gaussian Mixture Models (GMM)

The Core Idea

GMM is K-Means's smarter sibling. Instead of assuming all clusters are spheres, GMM assumes each cluster follows a Gaussian (bell curve) distribution, which can be stretched into an ellipse of any orientation and size.

K-Means
"Each cluster is a ball."
Hard assignments: point belongs to exactly one cluster. Circular/spherical boundaries only.
GMM
"Each cluster is a Gaussian ellipse."
Soft assignments: point belongs to each cluster with some probability. Any elliptical boundary.

What is a Gaussian Distribution?

A Gaussian (normal distribution) is the classic bell curve. In 2D, it looks like a hill. GMM says each cluster is a hill — described by its center (mean) and its shape (covariance). The covariance controls whether the hill is round, elongated, or tilted.

⛰️ Landscape Analogy
Imagine data points are people standing on a landscape. Each cluster is a hill (Gaussian). People on top of a hill definitely belong to that cluster. People in the valley between two hills — GMM gives them probabilities: "60% on Hill A, 40% on Hill B." K-Means would force them to pick just one hill.

Soft vs. Hard Assignments

K-MeansGMM
Assignment typeHard — each point picks one clusterSoft — each point gets a probability for each cluster
OutputLabel: 0, 1, 2...Probabilities: [0.7, 0.2, 0.1] for 3 clusters
BoundarySharp lineGradual/fuzzy boundary
from sklearn.mixture import GaussianMixture

gmm = GaussianMixture(n_components=3, covariance_type='full', random_state=0)
gmm.fit(X)

labels = gmm.predict(X)           # Hard labels (argmax of probabilities)
probs = gmm.predict_proba(X)      # Soft: shape (n, K) — probability matrix

# Learned parameters
gmm.means_        # Shape (K, d) — K cluster centers
gmm.covariances_  # Shape depends on covariance_type

Covariance Types — Shape of Clusters

TypeWhat it meansFlexibilityRisk
sphericalAll clusters = circles (one radius each)Least flexibleMisses elongated clusters
diagAxis-aligned ellipses (no tilting)MediumMisses tilted clusters
tiedAll clusters share ONE covariance shapeMediumWrong if clusters have different shapes
fullEach cluster has its own full tilted ellipseMost flexibleCan overfit with few data points
💡 Which to Use?
full is the default/most powerful. Use simpler types if you have little data (to avoid overfitting) or if you know the clusters have similar shapes.

GMM Strengths & Weaknesses

✅ Strengths

  • Handles elliptical clusters of any orientation
  • Provides uncertainty (probabilities)
  • Can use BIC/AIC to choose n_components (unlike K-Means with just inertia)
  • Handles overlapping clusters gracefully

❌ Weaknesses

  • Assumes Gaussian shape — fails on rings or crescents
  • Slower than K-Means (uses EM algorithm)
  • Must still choose n_components (K)
  • Can overfit with full covariance on small datasets

Hierarchical Clustering

The Core Idea

Instead of partitioning into K clusters all at once, hierarchical clustering builds a tree of nested clusters. You can then "cut" the tree at any level to get different numbers of clusters — no need to commit to K upfront!

🌳 Family Tree Analogy
Think of a family tree. At the bottom: every person is their own "cluster." Moving up: siblings merge → families merge → extended families merge → the whole species is one cluster at the top. You can "cut" the tree at any generation to get different-sized groupings.

Two Approaches

Bottom-Up (Agglomerative) — More Common
Start with every point as its own cluster. Repeatedly merge the two closest clusters until one remains.
Top-Down (Divisive)
Start with all points in one cluster. Repeatedly split until every point is its own cluster.

The Dendrogram

A dendrogram is the tree diagram that shows the entire clustering history. The y-axis is the distance/dissimilarity at which two clusters were merged. The x-axis shows which points/clusters merged.

Distance | 5 | ┌────────────────────┐ | | | 3 | ┌───┤ ├───┐ | | | | | 1 | ┌──┤ ├──┐ ┌──┤ ├──┐ | | | | | | | | | | A B C D E F G H Cut at height 3 → 2 clusters: {A,B,C,D} and {E,F,G,H} Cut at height 1 → 4 clusters: {A,B}, {C,D}, {E,F}, {G,H}
💡 Reading a Dendrogram
The height of the horizontal line where two branches merge = how dissimilar those clusters were. Tall merges = you're forcing together very different groups. Look for the largest vertical gap — that's where you should "cut" for the natural number of clusters.

Linkage Criteria — How Do We Measure Cluster Distance?

When merging two clusters, how do you define the distance between them? Different definitions = different shapes of resulting clusters.

LinkageDistance = ...Characteristic
SingleDistance between the closest pair of points (one from each cluster)Can create long "chain" clusters; sensitive to outliers. Results in elongated clusters.
CompleteDistance between the farthest pair of pointsCreates compact, roughly equal-sized clusters. More robust.
AverageAverage distance between all pairs of pointsBalanced compromise between single and complete.
WardIncrease in total within-cluster variance after mergingMost similar to K-Means. Tends to create equally-sized clusters. Often best in practice.
🌉 Bridge Analogy for Linkage
Two groups of people on islands. Single = distance between the closest people on each island. Complete = distance between the farthest people. Average = average distance between all people. Ward = how much variance increases if everyone moves to one island.

Hierarchical vs. K-Means

PropertyK-MeansHierarchical (Agglomerative)
Must specify K?Yes, beforehandNo! Choose after seeing dendrogram
SpeedFast O(nKd)Slower O(n² log n) or O(n³)
ScalabilityScales to millionsStruggles with large n
Cluster shapesSpherical onlyDepends on linkage (can be flexible)
Reproducible?No (random init) unless seed setYes — deterministic
VisualizationElbow/silhouette plotBeautiful dendrogram

PCA — Principal Component Analysis

Why Do We Need Dimensionality Reduction?

  • Correlated features — if height and weight are correlated, you're storing redundant info. PCA merges them.
  • Curse of dimensionality — in high dimensions, distances get weird and meaningless. Everything looks equally far apart.
  • Visualization — you can't plot 100D data; PCA gets it to 2D for plotting.
  • Noise removal — low-variance components often capture noise; dropping them cleans data.
  • Faster downstream models — fewer features = faster training.

The PCA Intuition

📷 Best Camera Angle Analogy
Photographing a 3D sculpture: some angles capture its shape well (high variance in the photo — lots of spread). Other angles show it "flat on" (low variance — it looks like a line). PCA finds the angle(s) that capture the MOST spread/variation. Component 1 = the best single angle. Component 2 = the next best angle perpendicular to the first.

Formal Goal

Find k orthogonal directions (principal components) in d-dimensional space such that projecting the data onto these directions preserves the maximum variance.

  • Orthogonal = at 90° to each other (like X and Y axes). This ensures components are uncorrelated.
  • Maximum variance = the directions that show the most "spread" in the data.

The Four Key Matrices — Fully Explained

MatrixShapeMeaning
Xn × dOriginal data. n = number of examples, d = number of original features.
Wk × dThe principal components — k "directions" in the original d-dimensional space. Each row is one PC.
Zn × kThe transformed/compressed data. Same n examples, now in k-dimensional space.
X̂n × dReconstruction — approximate recreation of X from Z. Same shape as X but with some info lost.
Z = X @ W.T     (compress) X̂ = Z @ W       (reconstruct)
🎓 Student Test Score Analogy
X = 100 students × 20 test scores (all subjects). W = recipe for making 3 "mega-subjects": [overall ability, math vs verbal lean, science vs arts lean]. Z = 100 students × 3 mega-scores. You've compressed 20 test scores into 3 informative numbers per student. X̂ = approximate re-creation of the 20 original scores from the 3 mega-scores.

Understanding n, d, k

  • n = number of rows in X = number of data points/examples (doesn't change!)
  • d = number of columns in X = number of original features = number of coordinates in each cluster center
  • k = number of rows in W = number of components you keep. Always k ≤ d.
💡 Shape Check
Always verify shapes: Z must be (n, k). W must be (k, d). Then Z@W gives (n,d) = same shape as X. ✓

Explained Variance — How Much Information Do We Keep?

pca.explained_variance_ratio_
# Example: array([0.4890, 0.3925, 0.0812, ...])
# PC1 = 48.9%, PC2 = 39.25%, PC3 = 8.12%
# Together: 48.9 + 39.25 = 88.15% with just 2 components!

Scree Plot

Plot explained variance ratio vs. number of components. Look for the "elbow" — the point where additional components explain little extra variance. Components after the elbow are often mostly noise.

Cumulative Explained Variance

Add up the ratios: 48.9%, 88.15%, 96.27%... Choose k where cumulative variance exceeds your threshold (often 90–95%).

💡 Practical Rule
If cumulative explained variance reaches 90% at k=5, use n_components=5. You're keeping 90% of information with only 5 features instead of d!

PCA vs. Linear Regression — An Important Distinction

Linear RegressionPCA
GoalPredict y from x (minimize vertical residuals)Find directions of max variance (minimize perpendicular distance)
OrientationVertical: "predict y given x"Perpendicular: "find the axis of max spread"
Supervised?Yes — needs labels yNo — only uses X
OutputPredictions for yComponents W, transformed Z
📐 Geometric Difference
Regression finds the line that minimizes distances measured straight down (vertically). PCA finds the line that minimizes distances measured perpendicular to the line (at a right angle). For the first PC, it's the line that points along the longest axis of the data cloud.

PCA Code — Full Reference

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# ALWAYS standardize first! PCA is scale-sensitive.
scaler = StandardScaler()
X_std = scaler.fit_transform(X)

pca = PCA(n_components=2, random_state=0)
Z = pca.fit_transform(X_std)      # Compress: shape (n, 2)
X_reconstructed = pca.inverse_transform(Z)  # Reconstruct: shape (n, d)

# Key attributes
pca.explained_variance_ratio_    # Per-component fraction of variance
pca.components_                  # W matrix: shape (k, d) — the directions
pca.explained_variance_          # Raw variance per component (not ratio)
pca.n_components_                # k
pca.mean_                        # Mean of X (subtracted during transform)

# To reconstruct manually:
X_hat = Z @ pca.components_ + pca.mean_

# To reconstruct single example i:
Z[i, :] @ pca.components_

Why PCA is "Linear" + Information Loss

Linear: Each principal component is a weighted sum of original features. No x², no x×y — just linear combinations. The transformation Z = X @ W.T is a matrix multiplication = linear operation.

Information Loss: When k < d, you discard dimensions k+1 through d. The variance in those directions is lost. Setting k = d preserves all information (but reduces nothing). The X̂ you get from Z @ W is an approximation of X.

⚠️ Must Center (Standardize)
PCA is variance-based. Variance is measured around the mean. If data isn't centered, PCA's first component might just point toward the mean (a global offset), not the actual direction of spread. sklearn's PCA centers automatically. TruncatedSVD (used for sparse data) does NOT center.

SVD — Singular Value Decomposition

What is SVD?

SVD is the mathematical engine that powers PCA (and much more). Any matrix X can be exactly decomposed as:

X = U Σ Vᵀ
MatrixShapeMeaning
Un × n"Left singular vectors" — directions in example space. Columns are orthogonal unit vectors.
Σ (Sigma)n × d (diagonal)"Singular values" on the diagonal, all non-negative, sorted largest to smallest. Represents importance of each direction.
Vᵀd × d"Right singular vectors" — directions in feature space. Rows are the principal components (= rows of W in PCA).
🎭 Theater Analogy
Imagine a movie can be described by: (U) how each audience member relates to underlying "movie archetypes," (Σ) how important each archetype is (blockbuster vs. niche), (V) how each movie archetype relates to genres/features. SVD finds these hidden archetypes automatically!

Truncated SVD — The Compressed Version

Full SVD computes ALL components. Truncated SVD keeps only the top k singular values/vectors (the most important ones). This gives the best rank-k approximation of X.

X ≈ Uₖ Σₖ Vₖᵀ

Where the subscript k means we keep only the top k columns/rows.

How PCA relates to SVD

  • sklearn's PCA internally uses SVD on the centered data
  • The principal components (rows of W) are the rows of Vᵀ
  • The singular values in Σ relate to explained variance

When to Use TruncatedSVD vs. PCA

PCATruncatedSVD
Centers data?Yes (automatically)No
Best forDense data matricesSparse matrices (text, TF-IDF)
Use caseImages, tabular dataDocument-term matrices (LSA)
⚠️ Sparse Data Warning
For TF-IDF matrices (text data), you CANNOT center the data (it would destroy sparsity — filling billions of zeros with non-zero values). TruncatedSVD handles this correctly. PCA would be catastrophic on sparse data!

NMF — Non-negative Matrix Factorization

The Core Idea

NMF decomposes X ≈ W × H where all values in W and H must be ≥ 0. This non-negativity constraint forces parts-based, additive representations.

X ≈ W × H    (all entries ≥ 0)
MatrixShapeMeaning
Xn × dOriginal non-negative data
Wn × kActivations — how much each example uses each "part"
Hk × dComponents/basis — the "parts" (like PCA's W matrix)
🧩 LEGO Bricks Analogy
Imagine faces are built from LEGO parts: eyebrows, eyes, nose, mouth. W says "use 3 eyebrow bricks, 2 eye bricks, 1 nose brick" for this face. H defines what each brick looks like. You can ONLY add bricks — you can't "subtract" a nose. This is what non-negativity does: everything is additive, interpretable, and physically meaningful.

PCA vs. NMF — Complete Comparison

PropertyPCANMF
Data constraintAny data (pos or neg)X must be non-negative (≥ 0)
Component valuesPositive AND negativeAll positive (additive)
OrthogonalityComponents always orthogonal (90°)Components generally NOT orthogonal
OrderingPC1 > PC2 > ... by variance explainedNo natural ordering of components
InterpretationCan be abstract (positive + negative loadings)Usually interpretable as "parts" (additive)
UniquenessUnique solutionNon-unique — depends on initialization
Best forGeneral compression & visualizationImages (pixels), text (counts), biology (gene expression)
💡 When to Use NMF
When your data is naturally non-negative AND you want interpretable "parts":
  • Pixel intensities (0–255) — faces = sum of facial features
  • Word counts or TF-IDF — documents = mixture of topics
  • Gene expression levels — diseases = combinations of gene patterns

NMF in Practice — Topic Modeling

NMF is often used for topic modeling: discovering hidden topics in text. Each document = mixture of topics (row of W). Each topic = mixture of words (row of H). You can inspect H to see what words define each topic.

from sklearn.decomposition import NMF
from sklearn.preprocessing import MinMaxScaler

# Scale to [0,1] — NMF needs non-negative input
X_scaled = MinMaxScaler().fit_transform(X_nonneg)

nmf = NMF(n_components=5, init='nndsvda', random_state=0, max_iter=2000)
W = nmf.fit_transform(X_scaled)   # Activations: shape (n, 5)
H = nmf.components_               # Parts/topics: shape (5, d)

# For text topic modeling:
# W[i] = how much document i belongs to each topic
# H[j] = the word weights for topic j (inspect top words!)

LSA — Latent Semantic Analysis

What Is LSA?

LSA (Latent Semantic Analysis) is applying TruncatedSVD to a document-term matrix (like TF-IDF). It discovers hidden "topics" or semantic dimensions in text data.

📚 Library Analogy
You have 10,000 books and want to group them by topic. But two books on "machine learning" might use different words: one says "neural networks," another says "deep learning." LSA discovers that these words appear in the same contexts and are semantically related — even if they don't co-occur! It finds the hidden topics underlying the words.

The Pipeline

1
Get text — a collection of documents
2
TF-IDF — convert to a sparse document-term matrix X (n docs × d words)
3
TruncatedSVD (LSA) — reduce to k dimensions (topics). Result: Z matrix (n × k)
4
Use Z — cluster documents, find similar documents, visualize topics
from sklearn.decomposition import TruncatedSVD
from sklearn.feature_extraction.text import TfidfVectorizer

# Step 1: Create TF-IDF matrix (sparse!)
tfidf = TfidfVectorizer(max_features=5000)
X_tfidf = tfidf.fit_transform(documents)  # Shape: (n_docs, 5000)

# Step 2: LSA = TruncatedSVD on TF-IDF
lsa = TruncatedSVD(n_components=50, random_state=0)
Z = lsa.fit_transform(X_tfidf)  # Shape: (n_docs, 50)

# Now Z is a dense 50-dim "topic" representation of each doc
# Use it for clustering, similarity, visualization

Why NOT PCA for Text?

TF-IDF matrices are extremely sparse (mostly zeros — a document uses only a tiny fraction of all words). PCA centers the data first (subtracts the mean), which destroys sparsity. TruncatedSVD skips centering and works directly on sparse matrices.

TF-IDF — Quick Reminder

TF = Term Frequency: how often a word appears in THIS document. IDF = Inverse Document Frequency: how rare the word is across ALL documents. Common words like "the" get low IDF (appear everywhere, not informative). Rare but topic-specific words get high IDF.

TF-IDF(word, doc) = TF(word, doc) × log(N / df(word))

N = total documents, df(word) = number of documents containing the word.

Autoencoders — Neural Network Dimensionality Reduction

What Is an Autoencoder?

An autoencoder is a neural network that learns to compress data into a smaller representation and then reconstruct it. It has two parts:

🔵 Encoder
Takes the input (e.g., 784-pixel image) and compresses it into a small "latent space" (e.g., 32 numbers). Like PCA but with non-linear layers.
🟡 Decoder
Takes the compressed representation and tries to reconstruct the original input. Trained to minimize reconstruction error.
Input (784) → [Encoder layers] → Bottleneck (32) → [Decoder layers] → Output (784) ↑ "Latent space" or "Embedding" = compressed representation
📦 Shipping Analogy
You need to mail a sofa internationally but can only send a small box. The encoder "disassembles" the sofa into compact instructions that fit in the box. The decoder "reassembles" it from those instructions. The quality of reconstruction depends on how good the compression is. The "instructions in the box" = the latent representation.

💡 When Autoencoders Win
For images, audio, or any data with complex non-linear structure (e.g., faces, speech), autoencoders learn much better representations than PCA. PCA for faces would give abstract linear combinations; an autoencoder might learn to represent "face angle," "lighting," "expression" etc. in its latent dimensions.

Scaling — When and How to Scale Your Data

Why Scaling Matters

Many algorithms use distances (Euclidean distance). If feature 1 is in kilometers (range 0–10,000) and feature 2 is in grams (range 0–1), distances will be dominated by feature 1. Feature 2 effectively gets ignored!

📏 Ruler Analogy
Measuring similarity between cities: one axis in km (Moscow to Tokyo = 7,500), another in degrees latitude (difference = 20). The km value utterly dominates. Scaling puts them on equal footing — "number of standard deviations from mean."

The Two Main Scalers

ScalerFormulaOutput RangeUse When
StandardScaler(x − mean) / stdMean=0, Std=1 (unbounded)General purpose. Use before PCA, K-Means, DBSCAN. Handles outliers reasonably.
MinMaxScaler(x − min) / (max − min)[0, 1] exactlyWhen you need truly non-negative output (NMF!). Sensitive to outliers.
